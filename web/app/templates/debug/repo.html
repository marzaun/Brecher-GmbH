Debug-Service Konfiguration

Der Debug-Modus wird über den Binary-Service (Port 9999) aktiviert.

Aktivierungs-PIN

# Debug-PIN für den Diagnose-Service
# Wird im Debug-Panel des Webportals eingegeben
DEBUG_SERVICE_PIN=7834

Verfügbare Befehle (Binary-Service)

# Verbindung: nc <host> 9999
PING              - Verbindungstest
DEBUG <PIN>       - Debug-Modus aktivieren (PIN: siehe oben)
AUTH <user> <otp> - Admin-Authentifizierung + Diagnose
QUIT              - Verbindung beenden

Authentifizierung

OTP-Token (Admin-Zugang)

Für den Admin-Login wird ein zeitbasiertes OTP benötigt. Der Token wird sowohl vom Web-Frontend als auch vom Binary-Service validiert.

Algorithmus

# Schritt 1: Zeitschritt berechnen (5-Minuten-Fenster)
time_step = unix_timestamp // 300

# Schritt 2: Eingabestring zusammensetzen
raw = "{username}:BRECHER_GEHEIM_2026:{time_step}"

# Schritt 3: SHA-256 Hash berechnen
digest = sha256(raw).hexdigest()

# Schritt 4: Erste 6 Ziffern aus dem Hex-Digest extrahieren
otp = [nur Ziffern aus digest][:6]

# Schritt 5: Falls weniger als 6 Ziffern, mit '0' auffüllen

Beispiel-Implementierung (Python)

import hashlib, time

def generate_otp(username):
    time_step = int(time.time()) // 300
    raw = f"{username}:BRECHER_GEHEIM_2026:{time_step}"
    digest = hashlib.sha256(raw.encode()).hexdigest()
    otp = ''.join(c for c in digest if c.isdigit())[:6]
    return otp.ljust(6, '0')
